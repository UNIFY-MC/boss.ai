# Story 1.4: Logo Build Day → SVG inline + `loading="lazy"` invertido

**Epic:** epic-001-tech-debt-remediation
**Status:** Draft
**Wave:** 1
**TD source:** TD-004 (+ QW-UX-3 absorvido)
**Severity:** HIGH
**Effort:** S (1h)

## Goal

Substituir o logo Build Day (PNG 788 KB) por **SVG inline** no `<head>` partilhado das 5 páginas, e inverter o uso de `loading="lazy"` — eager nos elementos críticos do LCP (logo, hero), lazy só nas imagens decorativas abaixo da fold.

## Acceptance Criteria

- [ ] AC1: Logo Build Day re-exportado como SVG < 5 KB (idealmente <2 KB), inline no HTML (não `<img src>`)
- [ ] AC2: SVG aparece no `<head>` ou início do `<body>` das 5 páginas Build Day (partial partilhado se existir, senão duplicado coerentemente)
- [ ] AC3: PNG `logo.png` (788 KB) **removido** do repositório (ou movido para `_assets-archive/` se houver razão para guardar)
- [ ] AC4: Em todas as 5 páginas Build Day, audit dos `<img loading="lazy">`: imagens acima da fold (LCP candidates) usam `loading="eager"` ou default (sem atributo); só imagens decorativas abaixo da fold mantêm `loading="lazy"`
- [ ] AC5: Lighthouse Performance score (mobile 3G simulado) > 80 em pelo menos 3 das 5 páginas
- [ ] AC6: LCP < 2.5s em 5/5 páginas (medido manualmente ou via Lighthouse)
- [ ] AC7: SVG aceita `currentColor` (permite tematização futura via design system Wave 2)
- [ ] AC8: Smoke test visual: logo renderiza idêntico ao PNG original em browser desktop e mobile

## Tasks

- [ ] Task 1: Re-exportar o logo original como SVG optimizado (ferramenta: SVGO ou export do Figma/Illustrator)
- [ ] Task 2: Verificar tamanho < 5 KB; se maior, simplificar paths
- [ ] Task 3: Inserir SVG inline no markup das 5 páginas Build Day (template partilhado se existir)
- [ ] Task 4: Auditar todos `<img>` das 5 páginas: identificar quais estão no viewport inicial
- [ ] Task 5: Para imagens acima da fold: remover `loading="lazy"` ou substituir por `loading="eager"`
- [ ] Task 6: Para imagens decorativas abaixo da fold: garantir `loading="lazy"`
- [ ] Task 7: Remover `claude-code-build-day/assets/logo.png` do repositório
- [ ] Task 8: Validar com Lighthouse e medir LCP em 5/5 páginas

## Out of scope

- Optimização de outras imagens das outras formações
- Implementação do design system com tokens (Wave 2 Story 2.1)
- Lighthouse CI no pipeline (Wave 4 Story 4.2)
