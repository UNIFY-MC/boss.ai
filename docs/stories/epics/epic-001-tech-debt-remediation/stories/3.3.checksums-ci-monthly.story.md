# Story 3.3: CI mensal de recompute de checksums

**Epic:** epic-001-tech-debt-remediation

**Severity:** MEDIUM

## Goal

Adicionar step CI **scheduled** (cron mensal) que corre `aiox registry --recompute-checksums --dry-run` e abre issue automaticamente se houver drift. Garante que os 821 checksums do `entity-registry.yaml` se mantêm verificados ao longo do tempo.

## Acceptance Criteria

- [ ] AC1: Existe workflow GitHub Actions em `.github/workflows/registry-checksums.yml` com trigger `schedule: cron: '0 0 1 * *'` (1º dia do mês, meia-noite UTC)
- [ ] AC2: O job corre `aiox registry --recompute-checksums --dry-run` (ou equivalente) e captura output
- [ ] AC3: Se houver drift detectado, o job usa `gh issue create` para abrir issue automaticamente com label `tech-debt/checksum-drift` e corpo descritivo (lista de checksums divergentes)
- [ ] AC4: Se não houver drift, o job termina com exit 0 sem criar issue
- [ ] AC5: Workflow tem manual trigger (`workflow_dispatch`) para execução on-demand
- [ ] AC6: Teste manual: trigger via `workflow_dispatch` em PR validation funciona

## Tasks

- [ ] Task 1: Criar `.github/workflows/registry-checksums.yml`

- [ ] Task 3: Implementar lógica condicional: se drift → criar issue
- [ ] Task 4: Adicionar `workflow_dispatch`
- [ ] Task 5: Testar via dispatch manual
