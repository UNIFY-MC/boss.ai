# Story 2.1: Design system hybrid — `_shared/tokens.css` + `_shared/themes/{nome}.css`

**Epic:** epic-001-tech-debt-remediation
**Status:** Draft
**Wave:** 2
**TD source:** TD-008 (decisão central de Uma — Phase 6)
**Severity:** MEDIUM (mas **cascade unlocker** — habilita 4 outros TDs)
**Effort:** L (3-4h)

## Goal

Implementar o **hybrid approach** decidido no Phase 8 do Brownfield Discovery: **preservar identidades visuais** das 4 sub-formações como valor arquivístico, mas **unificar o contrato semântico** num conjunto partilhado de tokens. Cada nova formação passa a ser **1 ficheiro de theme**, sem inventar vocabulário novo.

Componentes partilhados (`sidebar.css`, `page-toc.css`) deixam de ler tokens específicos de formação e passam a consumir **apenas** tokens semânticos de `_shared/tokens.css`.

## Acceptance Criteria

- [ ] AC1: Existe `_shared/tokens.css` com vocabulário **semântico** mínimo:
  - `--accent` (cor primária da formação)
  - `--accent-muted` (variante esbatida)
  - `--bg-base` (fundo principal)
  - `--bg-surface` (cartões, sidebar)
  - `--text-primary`
  - `--text-secondary`
  - `--font-display` (display/headings)
  - `--font-body`
  - `--font-mono`
  - `--border-color`
  - `--shadow-elevation-1` (sutil)
  - `--shadow-elevation-2` (cards)
  - `--radius-sm`, `--radius-md`, `--radius-lg`
- [ ] AC2: Existe `_shared/themes/legora.css`, `_shared/themes/aiox-limon.css`, `_shared/themes/kopkai-cyan.css` (e theme do Build Day) — cada um override apenas dos tokens semânticos
- [ ] AC3: Cada página de formação inclui `_shared/tokens.css` + o seu theme específico via `<link>` (ordem importa)
- [ ] AC4: `_shared/sidebar.css` e `_shared/page-toc.css` reescritos para usar **apenas** os tokens semânticos (nenhuma referência a `--legora-*`, `--kopkai-*`, etc.)
- [ ] AC5: Visualmente, cada formação renderiza idêntica (ou superior) ao estado actual — smoke test em 11/11 páginas
- [ ] AC6: Adicionar uma **nova formação fictícia** ("test-theme") deve requerer apenas: criar `_shared/themes/test-theme.css` com 12-15 overrides — sem editar nada em `_shared/tokens.css` nem componentes partilhados (validação do critério escalabilidade)
- [ ] AC7: Sem build step adicionado — continua a funcionar como HTML estático
- [ ] AC8: `_shared/tokens.css` inclui também `:focus-visible` global (preparação para Story 2.2)
