# Story 3.4: Testes unitários para hooks `.cjs` + loaders de YAML

**Epic:** epic-001-tech-debt-remediation
**Status:** Draft
**Wave:** 3
**TD source:** TD-003 (scope alargado por Dara)
**Severity:** HIGH (parte do safety net global)
**Effort:** M-L (1-2 dias)

## Goal

Cobrir os 2 hooks `.cjs` críticos (`enforce-git-push-authority.cjs`, `synapse-engine.cjs`) e os loaders consumidores dos YAMLs validados em Story 3.1 com testes unitários. Hoje não há `tests/` no projecto — esta story estabelece a infra de testing E os primeiros testes.

## Acceptance Criteria

- [ ] AC1: Existe directoria `tests/` na raiz com infra de testing configurada (Jest, Vitest ou equivalente — confirmar com `core-config.yaml` ou @architect)
- [ ] AC2: `enforce-git-push-authority.cjs` tem cobertura ≥ 80%:
  - Block quando agent != @devops e comando contém `git push`
  - Block quando comando contém `gh pr create` ou `gh pr merge`
  - Allow quando agent == @devops
  - Edge cases: `git push --force`, `git push --no-verify`
- [ ] AC3: `synapse-engine.cjs` tem cobertura ≥ 70% (foco em paths críticos: agent activation, context injection)
- [ ] AC4: Loaders dos YAMLs em `.aiox-core/data/` têm testes que:
  - Carregam o YAML real (smoke test)
  - Validam contra o schema (Story 3.1)
  - Cobrem erros esperados (ficheiro malformado, campo obrigatório em falta)
- [ ] AC5: Comando `npm test` corre todos os testes e exit 0 em sucesso
- [ ] AC6: CI step (em `.github/workflows/ci.yml`) corre `npm test` e bloqueia merge em falha
- [ ] AC7: README ou `tests/README.md` documenta como correr e escrever testes

## Tasks

- [ ] Task 1: Decidir framework (Vitest recomendado para alinhamento moderno) — consultar @architect
- [ ] Task 2: Setup inicial: `package.json` test script, config do framework, primeiro teste smoke
- [ ] Task 3: Escrever testes para `enforce-git-push-authority.cjs` (com mocks de stdin/stdout/exit)
- [ ] Task 4: Escrever testes para `synapse-engine.cjs` (foco em paths críticos)
- [ ] Task 5: Escrever testes para loaders dos YAMLs (1 teste por YAML mínimo)
- [ ] Task 6: Adicionar step `npm test` no CI
- [ ] Task 7: Documentar em `tests/README.md`

## Out of scope

- E2E tests
- Visual regression tests do acervo
- Cobertura 100% (alvo realista 70-80%)
