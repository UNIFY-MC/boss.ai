# Story 2.4: `prefers-color-scheme` global no acervo (10 páginas de formações)

**Epic:** epic-001-tech-debt-remediation
**Status:** Draft
**Wave:** 2
**TD source:** TD-015
**Severity:** MEDIUM
**Effort:** M (1-2h)

## Goal

Estender o suporte a `prefers-color-scheme: dark` (estabelecido em 2.3 para o hub) a **todas as outras 10 páginas** do acervo. Cada formação mantém a sua identidade visual, mas ganha variante escura coerente quando o utilizador a tem activa no SO.

## Acceptance Criteria

- [ ] AC1: Cada um dos 4 themes em `_shared/themes/{nome}.css` (Story 2.1) tem bloco `@media (prefers-color-scheme: dark) { ... }` com overrides apropriados aos tokens semânticos
- [ ] AC2: Em todas as 11 páginas do acervo, alternar `prefers-color-scheme` no DevTools renderiza variante apropriada (sem flash, sem JS)
- [ ] AC3: Contraste WCAG AA em todas as variantes claras e escuras
- [ ] AC4: A identidade visual de cada formação é **preservada** em dark mode (e.g., KopkAI continua KopkAI mesmo em escuro — accent ciano mantido)
- [ ] AC5: Logos/imagens que precisem de variante dark são tratados (via `currentColor` para SVGs, ou `<picture>` com `media` para PNGs)
- [ ] AC6: Lighthouse a11y score ≥ 95 em variante dark (validação manual em DevTools)

## Tasks

- [ ] Task 1: Para cada theme em `_shared/themes/`, decidir paleta dark-mode (consulta com Mário se preciso)
- [ ] Task 2: Adicionar `@media (prefers-color-scheme: dark)` block a cada theme
- [ ] Task 3: Auditar imagens e logos por formação — tratar com `currentColor` ou `<picture>`
- [ ] Task 4: Smoke test em 10/10 páginas em ambos os modos
- [ ] Task 5: Validar contraste WCAG AA em ambas as variantes
