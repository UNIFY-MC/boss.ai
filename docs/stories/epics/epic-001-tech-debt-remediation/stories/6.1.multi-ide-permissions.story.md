# Story 6.1: Equivalência de permissions multi-IDE

**Epic:** epic-001-tech-debt-remediation

**Effort:** M (1 dia)

## Goal

`aiox ideSync --check` (Story 4.1) verifica sincronização de ficheiros entre 7 directorias de IDE, mas **não** verifica equivalência semântica de permissions/deny rules entre IDEs. Resultado: boundary L1/L2 (framework vs project) só está protegida em Claude Code; em IDEs onde não foi replicado, um `@dev` pode editar `.aiox-core/core/` sem block.

Estender `aiox ideSync --check` para validar equivalência semântica de:
- Deny rules
- Allow rules
- Hook registration (PreToolUse / PostToolUse)
- Agent activation patterns
