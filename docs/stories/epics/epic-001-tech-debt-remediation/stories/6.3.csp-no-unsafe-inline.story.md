# Story 6.3: CSP sem `'unsafe-inline'` (build step Tailwind + extrair scripts inline)

**Epic:** epic-001-tech-debt-remediation
**Status:** Draft (**requer sign-off do Mário antes de arrancar**)
**Wave:** 6 (Strategic refactors)
**TD source:** TD-011
**Severity:** MEDIUM (mas Uma recomenda "aceite, não resolver" até trigger concreto)
**Effort:** M (1 dia)

## Goal

Hoje o acervo usa Tailwind CDN, o que força `'unsafe-inline'` em `script-src` e `style-src` da CSP. Para apertar a CSP, seria preciso:
- Pré-compilar Tailwind (build step) → contraria o princípio "sem build step no acervo"
- Extrair scripts inline para ficheiros externos
- Eventualmente: nonce ou hash-based CSP

**Recomendação de Uma:** "aceite, não resolver" até aparecer trigger concreto (e.g., formulário de contacto, conteúdo gerado por user). Hoje o acervo é privado, não tem input dinâmico — o risco prático é baixo.

## Decisão estratégica requerida

**Antes de arrancar:** Mário deve responder:
- Há trigger concreto? (e.g., adicionar formulário, abrir acervo a público externo?)
- Vale o esforço de M (1 dia) + custo de manutenção do build step?
- Aceitar `'unsafe-inline'` como decisão arquitectural permanente vs temporária?
