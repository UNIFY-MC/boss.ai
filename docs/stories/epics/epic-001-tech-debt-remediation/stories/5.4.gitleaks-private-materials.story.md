# Story 5.4: Pre-commit/gitleaks rule para materiais privados de terceiros

**Epic:** epic-001-tech-debt-remediation
**Status:** Draft
**Wave:** 5
**TD source:** TD-017
**Severity:** LOW (mas risco legal/IP material)
**Effort:** XS (30 min)

## Goal

Hoje os materiais privados (slides PDF, vídeos, materiais de formações pagas) estão protegidos apenas por path-exclusion em `.gitignore` / `.vercelignore`. Um erro humano (mover ficheiro para fora do path ignorado) pode levar à publicação acidental.

Adicionar uma camada extra: pre-commit hook ou gitleaks rule custom que detecta materiais privados de terceiros **fora dos paths esperados** e bloqueia o commit.

## Acceptance Criteria

- [ ] AC1: Existe regra (pre-commit hook OU gitleaks custom rule) que detecta:
  - Ficheiros de formações específicas (e.g., `Legora_*.pdf`, `KopkAI_*.mp4`) fora dos paths em `.gitignore`
  - Padrões de nomes de instructores de formação (configurável)
- [ ] AC2: A regra bloqueia o commit local (pre-commit) E falha o CI
- [ ] AC3: Configuração documentada em `.gitleaks.toml` (ou `.husky/pre-commit`) com comentários explicativos
- [ ] AC4: Teste deliberado: criar ficheiro com nome flagado fora de path ignorado → commit bloqueado
- [ ] AC5: Falsos positivos são <5% (não bloqueia commits legítimos do framework AIOX)
