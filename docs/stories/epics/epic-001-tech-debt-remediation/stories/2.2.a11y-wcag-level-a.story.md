# Story 2.2: Acessibilidade WCAG Level A — skip-link + `<main>` + focus + `aria-current` + tap targets

**Epic:** epic-001-tech-debt-remediation
**Status:** Draft
**Wave:** 2
**TD source:** TD-019 (+ TD-027, TD-028, QW-UX-2 absorvidos)
**Severity:** MEDIUM (WCAG 2.1 Level A failure — compliance, não polish)
**Effort:** S+ (2-3h)

## Goal

Saneamento completo de acessibilidade nas 11 páginas do acervo, atingindo WCAG 2.1 Level A em todos os critérios actualmente falhados:
- `2.4.1 Bypass Blocks` (skip-link)
- `1.3.1 Info and Relationships` (`<main>` landmark)
- `2.4.7 Focus Visible` (`:focus-visible`)
- `2.5.5 Target Size` (tap targets ≥ 44×44 px)
- `2.4.4 Link Purpose` (`aria-current="page"` na sidebar)
- Keyboard nav (focus trap no drawer mobile + `Escape` fecha)

## Acceptance Criteria

- [ ] AC1: Em todas as 11 páginas, primeiro elemento focável é skip-link `<a href="#content">Ir para conteúdo</a>` (visualmente oculto até receber foco)
- [ ] AC2: Em todas as 11 páginas, o conteúdo principal está envolvido em `<main id="content">...</main>` (skip-link aponta para este `id`)
- [ ] AC3: `:focus-visible` definido globalmente em `_shared/tokens.css` (consumindo `--accent`) — outline coerente em todos os elementos interactivos
- [ ] AC4: Na sidebar partilhada, o item activo (página corrente) recebe `aria-current="page"` (lógica em `_shared/sidebar.js`)
- [ ] AC5: Drawer mobile (sidebar em viewport <1000px): ao abrir, foco trapped no drawer; `Tab` cicla dentro; `Shift+Tab` cicla em sentido inverso; `Escape` fecha o drawer
- [ ] AC6: Audit de tap targets: todos os elementos interactivos (links sidebar, filter chips, botões) têm área ≥ 44×44 px em viewport mobile (DevTools device mode)
- [ ] AC7: Audit de overflow em viewport <360px (iPhone SE): sem scroll horizontal involuntário em 11/11 páginas
- [ ] AC8: Lighthouse a11y score ≥ 95 em 11/11 páginas (medido manualmente nesta story; CI cobre em 4.2)
- [ ] AC9: Teste manual com leitor de ecrã (NVDA ou VoiceOver) — pelo menos 2 páginas validadas com nav por landmarks (`D` salta para `<main>`)

## Tasks

- [ ] Task 1: Adicionar skip-link em `_shared/` (ou inline no início de `<body>`) com CSS para `position: absolute; left: -9999px` até `:focus`
- [ ] Task 2: Envolver conteúdo principal de cada uma das 11 páginas em `<main id="content">` (auditar primeiro: páginas podem já ter `<div class="content">` etc.)
- [ ] Task 3: Adicionar `:focus-visible` global em `_shared/tokens.css` (já preparado em Story 2.1)
- [ ] Task 4: Em `_shared/sidebar.js`, adicionar lógica `if (currentPath === itemPath) link.setAttribute('aria-current', 'page')`
- [ ] Task 5: Implementar focus trap no drawer mobile (lista todos focáveis, `Tab`/`Shift+Tab` ciclam, `Escape` chama close handler)
- [ ] Task 6: Audit dos tap targets em DevTools mobile mode; aumentar padding/min-width/min-height onde necessário
- [ ] Task 7: Audit de overflow horizontal em viewport <360px; ajustar via `overflow-x: hidden` em containers ou refactor de elementos largos
- [ ] Task 8: Correr Lighthouse a11y em 11/11 páginas e iterar até score ≥ 95
- [ ] Task 9: Teste manual com leitor de ecrã (2 páginas mínimo)
