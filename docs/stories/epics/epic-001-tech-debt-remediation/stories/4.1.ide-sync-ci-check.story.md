# Story 4.1: CI step + pre-commit `aiox ideSync --check`

**Epic:** epic-001-tech-debt-remediation
**Status:** Draft
**Wave:** 4
**TD source:** TD-006
**Severity:** MEDIUM
**Effort:** S (1-2h)

## Goal

`core-config.yaml § ideSync` declara `strictMode: true` mas **nenhum step CI** executa `aiox ideSync --check`. Drift silencioso entre as 7 directorias de IDE config (`.claude/`, `.cursor/`, `.gemini/`, etc.) pode acumular sem alarme.

Adicionar:
1. CI step que falha o job em caso de drift
2. Pre-commit hook análogo (advisory) para detecção local

## Acceptance Criteria

- [ ] AC1: Existe step em `.github/workflows/ci.yml` (após linting, antes de testes) que corre `aiox ideSync --check`
- [ ] AC2: Em caso de drift, o step exit code != 0 e output lista exactamente quais ficheiros divergem entre quais IDEs
- [ ] AC3: Existe pre-commit hook que corre `aiox ideSync --check` e avisa (não bloqueia) o committer
- [ ] AC4: Documentação curta em `core-config.yaml` § ideSync ou em `.claude/rules/` explicando o fluxo: edit primário → `aiox ideSync` → commit
- [ ] AC5: Teste deliberado: introduzir drift entre 2 IDEs → CI falha → corrigir → CI passa

## Tasks

- [ ] Task 1: Verificar que `aiox ideSync --check` existe e funciona localmente
- [ ] Task 2: Adicionar step CI
- [ ] Task 3: Adicionar pre-commit hook (advisory)
- [ ] Task 4: Documentar fluxo de sync
- [ ] Task 5: Teste deliberado

## Out of scope

- Equivalência semântica de permissions/deny rules entre IDEs (Story 6.1)
- Auto-fix de drift
