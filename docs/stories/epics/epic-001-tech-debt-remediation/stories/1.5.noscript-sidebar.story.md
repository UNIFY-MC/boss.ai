# Story 1.5: `<noscript>` fallback na sidebar partilhada

**Epic:** epic-001-tech-debt-remediation
**Status:** Draft
**Wave:** 1
**TD source:** TD-025 (QW-UX-1 absorvido)
**Severity:** HIGH
**Effort:** XS (15 min)

## Goal

Adicionar um `<noscript>` fallback dentro do `<div id="sidebar-mount">` em todas as 11 páginas do acervo. A sidebar inteira é construída por `_shared/sidebar.js` — sem JS, hoje a navegação **desaparece**, quebrando o princípio "HTML estático manutenível por décadas".

Sem refactor do `sidebar.js`. Apenas fallback estático mínimo.

## Acceptance Criteria

- [ ] AC1: Em todas as 11 páginas do acervo, `<div id="sidebar-mount">` contém um `<noscript>` com lista mínima de links para todas as formações
- [ ] AC2: O `<noscript>` contém: link para hub principal + lista de formações (mínimo nome + link)
- [ ] AC3: Com JavaScript desactivado (DevTools → "Disable JavaScript"), a sidebar renderiza a versão fallback e é navegável
- [ ] AC4: Com JavaScript activo, o `_shared/sidebar.js` substitui o conteúdo do `<div id="sidebar-mount">` (o `<noscript>` desaparece — comportamento HTML padrão)
- [ ] AC5: O fallback é HTML semântico (lista `<ul>` com `<a>`), sem CSS específico necessário (herda dos estilos `_shared/sidebar.css` em estado básico)

## Tasks

- [ ] Task 1: Inventariar todas as páginas com `<div id="sidebar-mount">` (deveriam ser 11)
- [ ] Task 2: Definir lista canónica de formações + URLs (pode ler de `_shared/sidebar.js` config)
- [ ] Task 3: Construir bloco `<noscript>` com `<ul><li><a>` minimal
- [ ] Task 4: Inserir o `<noscript>` dentro de `<div id="sidebar-mount">` em 11/11 páginas
- [ ] Task 5: Smoke test: abrir uma página com JS desactivado, verificar links funcionam

## Out of scope

- Refactor do `_shared/sidebar.js` para SSR/static-first (separado, fora do scope brownfield)
- Estilização avançada do fallback
- Animações ou hover na versão fallback
