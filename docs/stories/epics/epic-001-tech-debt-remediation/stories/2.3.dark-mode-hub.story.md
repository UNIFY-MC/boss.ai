# Story 2.3: `prefers-color-scheme` no hub (variante opcional, claro como default)

**Epic:** epic-001-tech-debt-remediation
**Status:** Draft
**Wave:** 2
**TD source:** TD-010
**Severity:** MEDIUM
**Effort:** M (1-2h)

## Goal

Implementar suporte a `prefers-color-scheme: dark` no hub principal (`acervo-formacoes/index.html`), mas **mantendo claro como default**. Razão: o claro é um palate cleanser deliberado entre formações; dark mode é variante opcional para quem o tem activo no SO.

## Acceptance Criteria

- [ ] AC1: Hub renderiza em tema claro por defeito (cor de fundo clara, texto escuro) — sem alteração visual face ao estado actual
- [ ] AC2: Com `prefers-color-scheme: dark` activo no browser/SO, o hub renderiza variante escura coerente (fundo escuro, texto claro, acentos preservados)
- [ ] AC3: Os tokens semânticos de `_shared/tokens.css` (Story 2.1) ganham overrides dentro de `@media (prefers-color-scheme: dark) { :root { ... } }` no theme do hub
- [ ] AC4: Transição entre temas é imediata (sem flash) — usa apenas CSS, sem JS
- [ ] AC5: Contraste WCAG AA em ambas as variantes (claro e escuro)
- [ ] AC6: Logo/branding do hub legível em ambas as variantes (usa `currentColor` ou variantes por tema)
- [ ] AC7: Smoke test em DevTools "Rendering → Emulate CSS media feature prefers-color-scheme" (light + dark)

## Tasks

- [ ] Task 1: Identificar todos os elementos do hub que dependem de cor (fundo, texto, accent, bordas)
- [ ] Task 2: Mapear os valores dark-mode para cada token semântico (consulta com Mário)
- [ ] Task 3: Adicionar bloco `@media (prefers-color-scheme: dark) { :root { ... } }` no theme do hub (ou em CSS específico do hub)
- [ ] Task 4: Validar contraste WCAG AA em ambas as variantes
- [ ] Task 5: Smoke test em DevTools

## Out of scope

- Dark mode global no acervo (Story 2.4)
- Toggle manual de tema (decisão deliberada: apenas auto via OS)
