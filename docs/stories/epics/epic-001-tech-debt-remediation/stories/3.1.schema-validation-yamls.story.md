# Story 3.1: Schema validation runtime para YAMLs de `.aiox-core/data/`

**Epic:** epic-001-tech-debt-remediation
**Status:** Draft
**Wave:** 3
**TD source:** DS-001
**Severity:** MEDIUM (pré-requisito filesystem-first sustentável)
**Effort:** M (4-6h)

## Goal

Implementar validação runtime (JSON Schema ou Zod) para todos os ≥9 YAMLs em `.aiox-core/data/`:
- `entity-registry.yaml` (821 entidades — prioridade máxima)
- `workflow-patterns.yaml`
- `tool-registry.yaml`
- `workflow-chains.yaml`
- `agent-config-requirements.yaml`
- `mcp-tool-examples.yaml`
- `workflow-state-schema.yaml`
- `learned-patterns.yaml`
- `technical-preferences.yaml`

Validação corre em pre-commit hook + CI step. Edits inválidos passam a ser detectados **antes de runtime**.

## Acceptance Criteria

- [ ] AC1: Existe directoria `.aiox-core/schemas/` (ou equivalente) com 1 schema (JSON Schema ou Zod) por YAML em `data/`
- [ ] AC2: Cada schema cobre estrutura, tipos, campos obrigatórios, e (onde aplicável) **referential integrity** entre entidades (e.g., um `tool_id` em `workflow-patterns` deve existir em `tool-registry`)
- [ ] AC3: Existe CLI command `aiox validate-data` (ou `aiox doctor --validate-yamls`) que corre todos os schemas e reporta erros por ficheiro
- [ ] AC4: Pre-commit hook (em `.husky/` ou equivalente) corre `aiox validate-data` em ficheiros tocados em `.aiox-core/data/*.yaml`
- [ ] AC5: CI step (em `.github/workflows/ci.yml`) corre `aiox validate-data` em todos os ficheiros e falha o job em caso de erro
- [ ] AC6: Edição manual inválida (e.g., remover campo obrigatório de uma entidade do registry) é detectada localmente antes do commit
- [ ] AC7: Mensagens de erro são accionáveis (linha, campo, tipo esperado vs encontrado)
- [ ] AC8: Existe documentação curta (`.aiox-core/schemas/README.md`) descrevendo como adicionar/actualizar schemas

## Tasks

- [ ] Task 1: Decidir tooling: Zod (TS) vs Ajv (JSON Schema) — consulta `core-config.yaml` e padrões do projecto

- [ ] Task 3: Criar schemas para os restantes 8 YAMLs

- [ ] Task 7: Documentar em `.aiox-core/schemas/README.md`
