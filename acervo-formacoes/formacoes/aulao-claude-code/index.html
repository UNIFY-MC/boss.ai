Antes do como, o porquê

Pensar em tempos de Inteligência Artificial

A escrita reorganizou a memória (Platão já alertava sobre isso no Fedro), a imprensa reorganizou a autoridade do conhecimento, a internet reorganizou a busca. A IA reorganiza a síntese, que era, até então, a operação cognitiva considerada mais propriamente humana. A IA modifica o que entra no campo do pensável. Não apenas como pensamos, mas o material sobre o qual pensamos.

01

Hermenêutica em ambientes computacionais

Interpretar texto produzido por IA é ato hermenêutico novo. Exige habilidade própria: distinguir voz humana de saída sintética, identificar pegadas estilísticas de modelo. O texto da máquina é construído para soar correto, não para ser correto. A hermenêutica clássica supõe que o texto quer dizer algo; o texto da máquina simula querer dizer algo. O intérprete que aplica método hermenêutico clássico a saída de IA está interpretando uma superfície sem profundidade.

02

O que significa interpretar

Interpretar é um ato cognitivo de primeira ordem. O intérprete defronta-se com o texto bruto, mobiliza tradição interna — formação, leituras, experiência —, mobiliza método (gramatical, sistemático, teleológico, histórico) e constrói sentido em direção ao caso concreto. Ao final, assume autoria do sentido construído. O intérprete produz sentido. É um ato autoral no sentido forte do termo. Não por acaso a tradição jurídica sempre tratou o operador do Direito como figura quase-criativa: ele não aplica regra pronta, ele diz o direito. Jurisdição é, etimologicamente, juris-dictio — o ato de dizer o direito.

03

O que significa orquestrar

Orquestrar é um ato cognitivo de segunda ordem. O orquestrador não produz a matéria-prima — ele recebe outputs já processados de fontes diversas (IAs, bases de jurisprudência, agentes automatizados, assessores). Sua tarefa é avaliar, comparar, validar, integrar. Decidir o que entra, o que sai, o que se ajusta. Dar coerência ao conjunto. E, ao final, assumir responsabilidade pelo resultado. O orquestrador não é autor da matéria. É autor da composição. Compõe sentidos já produzidos, em vez de produzir sentido a partir do texto bruto.

04

A diferença é ontológica, não apenas operacional

Interpretar é trabalho vertical: do texto à mente do intérprete, da mente do intérprete ao caso. Orquestrar é trabalho horizontal: entre fontes que já entregam material pré-processado, exigindo seleção e integração. Aqui mora a tensão dogmática severa que praticamente ninguém está nomeando: a jurisdição, na tradição brasileira e ocidental, foi construída como ato interpretativo, não como ato orquestrador. Toda a teoria da motivação das decisões, da legitimidade democrática, da independência funcional, supõe um ato cognitivo vertical — o operador do Direito que se debruça sobre os autos e diz o direito, em primeira pessoa, com autoria. Se o operador do Direito vira orquestrador, ele não deixa de sê-lo. Mas a teoria do que é ser operador do Direito precisa ser refeita.

05

Dupla responsabilidade: governar sua própria IA

Se o operador do Direito é orquestrador-hermeneuta de segunda ordem, e se a sociedade decide algoritmicamente em escala crescente (crédito, contratação, justiça preditiva, vigilância, saúde), então ele tem dupla nova responsabilidade. Internamente: governar seu próprio uso de IA — não ser refém. Externamente: governar o uso de IA pela sociedade — julgar e atuar em casos onde algoritmos decidiram. Existem soluções no mercado que empacotam IA para o jurídico — e são ótimas, especialmente por serem acessíveis. Mas entendemos que existe um novo profissional em formação, um novo Direito em andamento, uma nova hermenêutica. Por isso faz sentido construir seus próprios frameworks e bases de conhecimento. Não existe almoço grátis.

Glossário essencial

Antes de instalar, entenda

Você não precisa ser programador. Mas precisa entender o terreno onde está pisando. São 4 conceitos que descomplicam tudo.

IDE (Ambiente de Desenvolvimento)

O terreno de construção

IDE é onde o código é escrito, editado e organizado. Pense como o terreno de uma obra — com todas as ferramentas à mão. VS Code e Cursor são IDEs. O Claude Code é o rádio comunicador que você usa dentro desse terreno para conversar com a IA.

Baixar VS Code →

Node.js

O motor da máquina

Node.js é o que permite rodar JavaScript fora do navegador. O Claude Code é construído com Node.js — sem ele, não funciona. É como o motor de um carro: você não precisa saber mecânica, mas precisa que esteja instalado.

Baixar Node.js →

Git

O Ctrl+Z profissional

Git salva todo o histórico do seu projeto. Cada mudança, cada versão, cada decisão. Se algo der errado, você volta no tempo. É controle de versão — funciona como uma linha do tempo do seu trabalho.

Baixar Git →

GitHub

Seu portfólio vivo

GitHub é onde seus projetos ficam publicados na nuvem. É o portfólio que empresas e colegas veem. Onde seu trabalho com IA fica registrado — a prova verificável das suas habilidades. Git é local, GitHub é na nuvem.

Criar conta no GitHub →

Mãos à obra

Pré-requisitos: o que instalar

Siga na ordem. Cada etapa depende da anterior. Se travar, a aula cobre tudo em detalhe.

1

Node.js (versão LTS)

Acesse nodejs.org e baixe a versão LTS (v20+ recomendado). Depois de instalar, verifique:

node --version
# Esperado: v18.x.x ou superior

npm --version
# Esperado: 9.x.x ou superior
2

Git

Acesse git-scm.com e instale. Depois configure (apenas uma vez):

git --version
# Esperado: git version 2.x.x

git config --global user.name "Seu Nome"
git config --global user.email "seu@email.com"
3

GitHub CLI

Acesse cli.github.com e instale. Depois autentique:

gh --version
# Esperado: gh version 2.x.x

gh auth login
# Siga as instruções na tela
4

Claude Code

Pré-requisito: assinatura Claude Pro (~US$20/mês). Instale e autentique:

npm install -g @anthropic-ai/claude-code

claude --version

claude
# Siga as instruções para autenticar

Documentação oficial: docs.anthropic.com/claude-code
