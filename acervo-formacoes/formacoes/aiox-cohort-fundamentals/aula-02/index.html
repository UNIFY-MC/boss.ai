Notas da aula

Resumo · aprendizagens

O AIOX é instalado dentro de cada projecto separado. Pensa assim: cada obra tem o seu próprio mestre de obras. Não se partilha o mesmo mestre entre obras diferentes.

Os 12 agentes

O AIOX tem 12 agentes, cada um com uma função específica:

Planeamento — pensa e organiza o projecto antes de qualquer código.
Desenvolvimento — constrói, testa e publica.
Orquestradores — coordenam todos os outros agentes.

Como chamar um agente

Usa-se a barra. Exemplo: /aiox-master. Nunca arroba — arroba é outra coisa. Para dar um comando dentro do agente, usa-se asterisco antes do comando.

Criar agente novo · passo que muita gente esqueceu

Depois de criar um agente, é preciso copiá-lo para a pasta .cloud/commands/agents. Sem isto, o agente existe mas não aparece para uso. Cria e copia, sempre os dois passos.

Chaves API e segredos

Password e chave de API vão sempre no ficheiro .env. Este ficheiro nunca sobe para o GitHub — é a gaveta trancada do projecto.
