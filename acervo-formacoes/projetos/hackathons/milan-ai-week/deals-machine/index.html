Deals Machine — The Autonomous Sales Agent

Agente de IA autónomo que corre todo o pipeline de vendas — desde o primeiro toque até ao follow-up de fecho — enquanto o humano só fala. Construído para founders solo e equipas pequenas que perdem deals por dívida de follow-up, não por qualidade de produto.

Problema · Solução

Inverte o rácio entre operação e conversa.

Problema

As equipas de vendas gastam a maior parte do tempo em trabalho operacional: registar chamadas, escrever follow-ups, actualizar campos de CRM, priorizar leads.

A venda efectiva — a conversa 1:1 humana — fica espremida entre tarefas. Deals Machine inverte esse rácio.

Solução

Ingere áudio das chamadas via Speechmatics, transcreve e analisa, sintetiza memória por lead que o agente relê antes de cada acção.

Planeia autonomamente o próximo passo: agenda follow-ups, redige emails personalizados, pontua o lead, mete call-backs em fila, actualiza o log de actividade. O humano só aparece e fala.

Pipeline autónomo

10 passos, do parágrafo ao email enviado.

Dá-se um parágrafo ao agente — "vendo SaaS de optimização de custos cloud a VPs de Engenharia de startups Series B/C com alto burn" — e ele corre o loop autónomo completo.

01 · ~60s

Constrói vertical

Claude redige o ICP, escolhe 4-5 fontes de sinal (RSS / Reddit / HN), define persona e voz do operador.

02

Raciocina sobre sinais

Scrape das fontes + web search com queries que o agente escolhe. Cada evento é pontuado por urgência via prompt vertical-specific.

03

Cadeias de consequência

Liga cada evento de alto sinal a um perfil de comprador com urgência nos próximos 30 dias. Sempre com citação.

04

Sourcing via Apollo.io

Filtragem server-side em títulos, indústrias, tamanho, departamentos, exclusões de concorrentes. Um contacto por empresa, senioridade mais alta.

05

Playbook executável

5 variantes de abertura, ângulos ordenados por recency-decay, cheat-sheet de objecções, pedidos, script de voicemail, brief pré-call.

06 · live

Coach em tempo real

Twilio Media Streams → Speechmatics RT → Haiku observa janela rolante de 30s e emite cards de coaching para o cockpit via Supabase Realtime.

07

Scoring pós-call

Speechmatics batch transcreve. Claude Sonnet produz scorecard estruturado: o que correu bem, o que melhorar, confirmações para o lead, sinal de outcome.

08

Loop de aprendizagem

Outcome tag ajusta pesos no cérebro (±0.5 closed_won → −0.2 killed). Playbook regenera com debounce de 30s. Próxima call usa script melhorado.

09

Follow-up auto

Drafted vertical-aware, com referência ao que aconteceu na call. Aparece em email_drafts.status='ready'.

10

Envio via Gmail real

One-click "Send now" na página de leads. OAuth garante endereço real do operador, respostas voltam à inbox correcta.

Arquitectura

Cockpit Next.js + worker Fastify em Vultr.

Next.js cockpitUI (Vercel) — verticals, pipeline, intelligence, leads, knowledge, analytics, settings
Fastify workerVultr Ubuntu 24.04 · Caddy · systemd — corre pipeline e playbook
SupabasePostgres + Realtime — estado, brain, drafts, eventos live
Twilio Media StreamsPipe áudio em tempo real para o worker
Speechmatics RT + BatchLive partials (1-2s) + transcrição batch enhanced com diarização
Claude Sonnet 4.6Geração de vertical, raciocínio, playbook, enrichment pós-call
Claude Haiku 4.5Coach loop (latência crítica) + classificador Lobster Trap
Apollo.ioSourcing de leads B2B com filtros server-side
Gmail OAuthEnvio do email a partir do endereço real do operador

Sponsor Tracks

Cada parceiro tem uma função real, não apenas selo.

Vultr

Cloud Compute

O worker Fastify corre numa instância Ubuntu 24.04 com systemd + Caddy reverse proxy (auto-WSS upgrades para Twilio Media Streams). Todo o reasoning, chamadas Apollo, bridge Twilio, jobs Speechmatics e classificações Lobster Trap fluem por Vultr.

Speechmatics

Call Transcription · Dual mode

Async batch (/jobs/) com diarização e operating_point: enhanced produz o transcript pós-call de alta fidelidade que alimenta scorecard e extracção de brain.

Real-time (WSS eu2.rt.speechmatics.com/v2, mulaw 8 kHz) faz pipe do áudio durante a chamada, com partials a chegar ao coach loop em 1-2 segundos. Auth via JWT cunhado on-demand.

Veea — Lobster Trap

Edge Security · Prompt-injection shield

Todo o input gerado por utilizador (chat, transcripts, documentos) passa por uma shield em camadas antes de chegar ao agente.

Lobster Trap container — serviço Fastify standalone com POST /check. Detecção em dois andares: 11 regex patterns (instantâneo, determinístico), depois classificador Claude Haiku 4.5. Inclui Dockerfile + systemd unit + guia de deploy.

Fallback in-process — se o container estiver inacessível, o worker faz fallback para o próprio regex + classifier. A shield nunca bloqueia o cockpit.

Anthropic — Claude

Modelos across the stack

Sonnet 4.6 — geração de vertical, chain reasoning, composição de script/playbook, enrichment pós-call.

Haiku 4.5 — classifier Lobster Trap, live coach loop (latency-critical, cadência de 5 segundos sobre janelas rolantes de transcript).
